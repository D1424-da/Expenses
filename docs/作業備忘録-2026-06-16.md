## 2. 支店名フィールドの追加

`store`（店名）とは別に `branch`（支店名・〇〇店）をデータに追加した。

- データ構造: `users/{uid}/expenses/{expenseId}` に `branch: string` を追加
- 入力フォーム（`static/index.html`）に「支店名」欄を追加
- 支出一覧・最安値比較に支店名を表示
- 最安値比較は**店名＋支店名ごと**に集計（同じチェーンでも別支店は別の店として比較）
- OCR（Gemini／Tesseract両方）で支店名を自動抽出
  - `static/app.js`（Geminiプロンプト）, `static/parser.js`, `app/parser.py`

※ 支店名は新フィールドのため、**既存データには入っていない**。今後保存する分から記録され、既存分は編集画面で手入力すれば比較に反映される。

## 3. Gemini APIキー漏洩問題

### 問題
APIキーを `static/firebase-config.js`（公開される静的ファイル）に直書きしていたため、
GitHubで公開され Google に「漏洩キー」として自動無効化された。
```
403 PERMISSION_DENIED: Your API key was reported as leaked. Please use another API key.
```

### 対応方針
**APIキーはフロントに置かず、バックエンドの環境変数に保持する**構成へ変更。

- フロントから直接Gemini呼び出しするコードを削除（`callGeminiOcr` 等）
- `firebase-config.js` から `GEMINI_API_KEY` / `USE_CLOUD_VISION` を削除
- フロントは `OCR_API_BASE` 経由でバックエンドを呼ぶだけ。失敗時はブラウザ内Tesseractにフォールバック

> ⚠️ 漏洩キーはコードからは消したが **git履歴には残る**（既にGoogleが無効化済みなので実害なし）。
> 新しいキーは絶対にコードに書かない。

## 4. バックエンド（OCRサービス）

### 構成
- `main.py` — FastAPI。`/api/ocr`（画像→構造化データ）, `/api/health`
- `app/gemini.py` — Gemini で画像から構造化抽出（新規追加）
- `app/ocr.py` / `app/parser.py` — Tesseract等のフォールバック用
- `OCR_ENGINE=gemini` のとき `/api/ocr` が Gemini の結果を返す

### Docker不要のネイティブ実行対応
- `main.py` で `ocr`（OpenCV依存）を**遅延import**化 → Geminiだけ使う軽量環境で起動が落ちない
- `requirements-gemini.txt` — fastapi / uvicorn / python-multipart のみ（OpenCV/Tesseract無し）
- `render.yaml` — Render Blueprint 用設定

## 5. Gemini APIキーの作り方（Cloud Shell）

Google Cloud Shell で実行（スクリプト同梱: `scripts/create-gemini-key.sh`）:

```bash
bash scripts/create-gemini-key.sh expenses-9af61
```

手動の場合:
```bash
gcloud config set project expenses-9af61
gcloud services enable generativelanguage.googleapis.com
gcloud services api-keys create --display-name="receipt-ocr-gemini" \
  --api-target=service=generativelanguage.googleapis.com
KEY_NAME=$(gcloud services api-keys list --filter="displayName=receipt-ocr-gemini" --format="value(name)" | head -n1)
gcloud services api-keys get-key-string "$KEY_NAME" --format="value(keyString)"
```

または [Google AI Studio](https://aistudio.google.com/apikey) でも発行可。

## 6. Render デプロイ手順（実施済み）

種類は **Web Service**（Static Site ではない点に注意）。

| 項目 | 値 |
|---|---|
| Runtime | Python 3 |
| Build Command | `pip install -r requirements-gemini.txt` |
| Start Command | `uvicorn main:app --host 0.0.0.0 --port $PORT` |
| Instance Type | Free |

環境変数（Renderの Environment タブ）:

| NAME | value |
|---|---|
| `OCR_ENGINE` | `gemini` |
| `GEMINI_MODEL` | `gemini-2.5-flash` |
| `GEMINI_API_KEY` | Cloud Shellで取得したキー（`AIza...`） |
| `CORS_ORIGINS` | `https://<GitHubユーザー名>.github.io` |

### デプロイ結果
- バックエンドURL: **https://expenses-ft54.onrender.com**
- 動作確認: `https://expenses-ft54.onrender.com/api/health` → `{"status":"ok","engine":"gemini"}`
- フロント `static/firebase-config.js` の `OCR_API_BASE` に上記URLを設定済み

> 💡 Renderの無料プランはアクセスが無いと一定時間でスリープし、次回の読み取りが十数秒遅くなる（仕様）。
